<ReaderGuide>

- **初学者** → L0〜L1（コンピュータは情報を 0/1 で持つ、アナログを «横に標本化・縦に量子化» してデジタル化する、$n$ bit で $2^n$ 通り）
- **一通り使える人** → L2（ナイキストの標本化定理 $f_s\ge2f_{\max}$、量子化の «1 bit ≈ 6 dB» の導出、接頭語のズレ、UTF-8 の可変長、ラスタ/ベクタ）
- **「完全に理解した」人** → L3 以降（誤り検出・訂正符号、データ圧縮、浮動小数点の内部表現、情報理論との接続は到達途上）

前提: とくになし（高校数学の指数・対数だけ）。ここで作る «bit＝情報量の単位» や標本化・量子化の見方は、[数値計算](/topics/numerical-computation)の[浮動小数点数](floating-point)や、データの «情報量» を測る情報理論（[KL ダイバージェンス](kullback-leibler-divergence)）の入り口になる。

</ReaderGuide>

<Level level={0} title="ブラックボックス（世界を 0 と 1 で持つ）">
  <Concept>

コンピュータは情報をすべて **0 と 1**（[ビット](bit)）で持つ。電圧の高/低のように «2 状態» が最も確実に区別できるからで、数も文字も画像も音も、突き詰めれば 0/1 の列にすぎない。

現実の «なめらかなアナログ» をコンピュータに取り込むには、2 つの方向で «飛び飛び» にする必要がある。

- **標本化（サンプリング）**：時間を等間隔に区切り、その瞬間の値だけを拾う（**横方向**の離散化）。
- **量子化**：拾った値を有限の段階（$n$ bit なら $2^n$ 段階）に丸める（**縦方向**の離散化）。

この «横に刻んで縦に丸める» がアナログ→デジタル変換（ADC）。どちらも細かくするほど原信号に近いが、そのぶんデータ量（ビット数）が増える。

  </Concept>
  <Interact>

下のラボで、アナログの正弦波（黒）を «デジタル化» する。$f_s$（標本化＝横の細かさ）と $n$ bit（量子化＝縦の細かさ）を動かすと、量子化後の階段波形（紫）が原波にどれだけ迫れるかが見える。$f_s$ を下げすぎると、標本点が波を追い切れず波形が «別の周波数» に化ける（エイリアシング）。

（答: 時間方向を飛び飛びにするのが **標本化（サンプリング）**、振幅方向を有限段階に丸めるのが **量子化**。前者は $f_s\ge2f_{\max}$、後者は $n$ bit で $2^n$ 段階が目安。両方あわせて ADC）

<Level level={1} title="古典的理解（2 進数・情報量の単位・論理演算）">
  <Concept>

**[2 進数](binary-number)は 0 と 1 だけで数を表す**。各桁は下から $1,2,4,8,\dots$（2 の冪）の重みを持つ：$1101_{(2)}=8+4+0+1=13$。10 進数を 2 進に直すには «2 で割り続けて余りを下から読む»（下のステッパー）。

**[ビット](bit)は «情報量の単位»**。1 bit は «0 か 1» の 2 択、$n$ bit なら $2^n$ 通りを区別できる。8 bit = 1 byte で $2^8=256$ 通り。逆に、等確率で $N$ 通りある «どれか» を知る情報量は $\log_2 N$ bit。

**[論理演算](logic-operation)は真(1)・偽(0)への演算**で、AND（両方 1 で 1）・OR（どちらか 1 で 1）・NOT（反転）・XOR（違えば 1）が基本。デジタル回路もビット演算（各桁への一括論理演算）も、この 0/1 の演算の上に立つ。

  </Concept>
  <Derivation title="なぜ «2 で割った余りを下から» 読むと 2 進数になるのか">

整数 $N$ を 2 で割ったときの商 $q$ と余り $r$ を考える（$r$ は 0 か 1）：

$$
N = 2q + r,\qquad r\in\{0,1\}.
$$

一方、$N$ の 2 進表記を $b_k b_{k-1}\cdots b_1 b_0$ とすると、定義より

$$
N = b_k 2^k + \cdots + b_1 2^1 + b_0 2^0 = 2\underbrace{(b_k 2^{k-1}+\cdots+b_1)}_{=\,q} + b_0.
$$

両式を見比べると、$2$ で割った**余り $r$ が最下位ビット $b_0$**、商 $q$ が «残りの桁 $b_k\cdots b_1$ を表す数» そのもの。だから $q$ に同じ操作を繰り返せば、$b_1, b_2, \dots$ が下の桁から順に決まる。商が 0 になったら終了し、余りを **下から上へ** 並べたものが 2 進表記になる。$2$ で割った余りは必ず 0 か 1 なので、各桁がちょうど 1 ビットに対応する。

  </Derivation>

<Level level={2} title="仮定体系（標本化定理・量子化の 6 dB 則・符号化）">
  <Concept>

デジタル化の «どこまで細かくすべきか» は、標本化と量子化それぞれに定量的な指針がある。

- **[標本化定理](sampling-theorem)（ナイキスト・シャノン）**: 原信号の最高周波数 $f_{\max}$ に対し $f_s\ge2f_{\max}$ で標本化すれば完全に復元できる。下回ると波が低い周波数に化ける **エイリアシング（折り返し）**。回る車輪が逆回転して見える動画やモアレがその例。
- **[量子化](quantization)の «1 bit ≈ 6 dB»**: $n$ bit は $2^n$ 段階、段幅 $\Delta=R/2^n$、誤差は最大 $\Delta/2$。bit を 1 増やすと $\Delta$ が半分＝誤差が半分になり、SN 比が $\mathrm{SNR}\approx6.02n+1.76$ dB で改善する（下で導出。CD の 16 bit で約 98 dB）。
- **接頭語のズレ**: 10 進の kilo $=10^3$ と 2 進の kibi $=2^{10}=1024$ が混用される。ズレは接頭語が大きいほど開き、テラ級で約 10%（«表示より容量が少ない» 問題）。
- **[文字符号化](character-encoding)**: 文字に番号（コードポイント）を与え、UTF-8 は ASCII を 1 バイト・日本語を 3 バイト…と **可変長** で表す。
- **[ラスタとベクタ](raster-vector)**: 画像は «画素の格子»（標本化＋量子化の結果）か «点・線・曲線の式» か。前者は拡大で粗く、後者は解像度に依らず滑らか。

いずれも «連続を有限のビットで近似する» という同じ発想の別の顔で、細かさ（精度）とデータ量（ビット数）のトレードオフが共通して現れる。

  </Concept>
  <Derivation title="なぜ 1 bit 増やすと量子化の SN 比が約 6 dB 改善するのか">

信号の振幅範囲を $R$、量子化ビット数を $n$ とすると、段幅は $\Delta=R/2^n$。量子化誤差 $e$ は各段の中で $[-\Delta/2,\ \Delta/2]$ にほぼ一様に散らばるとみなせる。一様分布の分散（＝雑音のパワー）は

$$
N = \mathbb{E}[e^2] = \frac{1}{\Delta}\int_{-\Delta/2}^{\Delta/2} e^2\,de = \frac{\Delta^2}{12}.
$$

信号のパワー $S$ は固定なので、SN 比をデシベルで書くと

$$
\mathrm{SNR}_{\mathrm{dB}} = 10\log_{10}\frac{S}{N}
= 10\log_{10}\frac{12S}{\Delta^2}
= 10\log_{10}\frac{12S}{(R/2^n)^2}.
$$

$2^n$ の項だけ取り出すと $10\log_{10}(2^{2n}) = 20n\log_{10}2 \approx 6.02\,n$。残り（$S,R$ に依る定数）を «フルスケール正弦波» で計算すると $+1.76$ dB になり、

$$
\boxed{\ \mathrm{SNR}\approx 6.02\,n + 1.76\ [\mathrm{dB}]\ }
$$

を得る。要点は $n\to n+1$ で $\Delta$ が半分＝雑音パワーが $1/4$ になり、$10\log_{10}4\approx6.02$ dB 改善すること——これが «1 bit ≈ 6 dB» 則。ラボでビットを 1 段上げるたびに SN 比が約 6 上がるのがそのまま見える。

  </Derivation>
  <Interact>

上のラボへ戻り、まず $n$ bit を上げて階段（紫）が原波に迫り SN 比が «1 bit ≈ 6 dB» で伸びるのを確かめ、次に $f_s$ をナイキスト下限 $2f_{\max}$ 未満まで下げて波形が化ける（エイリアシング）様子を見よう。横（標本化）と縦（量子化）は独立に効く。

  </Interact>
  <Exercise>

確認: 「16 bit で量子化した音（CD）の SN 比はおよそ何 dB か」——概算せよ。
（答: $6.02\times16+1.76\approx98$ dB。1 bit ごとに約 6 dB なので、16 bit で約 96＋α。8 bit なら約 50 dB、24 bit なら約 146 dB。ビットを増やすほど雑音が小さくなるがデータ量も増える）

  </Exercise>
