# クラス不均衡

分類問題で、あるクラス（多数派）のサンプル数が他のクラス（少数派）に比べて著しく多い状態。医療診断（陽性がまれ）、不正検知（不正取引がまれ）など、実務の分類問題の多くで発生する。

- **accuracy paradox**: «常に多数派を予測する» だけの分類器でも正解率（accuracy）は高く出る（例: 多数派95%なら正解率≈95%）。しかし少数派の再現率は0%——正解率だけでモデルを評価してはいけない典型例。
- 対処法は大きく2系統: データ側で調整する **[オーバーサンプリング](oversampling)**・**[アンダーサンプリング](undersampling)**（[SMOTE](smote)など）と、学習側で調整する **[コスト考慮型学習](cost-sensitive-learning)**（誤分類のコストをクラスごとに変える）。
- 評価指標も正解率ではなく[混同行列](confusion-matrix)から導かれる適合率・再現率・F値・AUCなど、少数派クラスに注目した指標を使う必要がある。
