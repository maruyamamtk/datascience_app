## 1. ビジョン / コンセプト

統計・機械学習・ベイズ・因果推論を、**数式を誤魔化さず**、かつ
**高校数学の素養があれば理解できる**形で、**手を動かして肌感覚を得ながら**学べるアプリ。

教科書（読む）でも問題集（解く）でもなく、**「概念 → 操作 → 演習」を1ページで往復**できる
インタラクティブな学習体験を中核に据える。

### コアの差別化要素
1. **数式の誠実さ**: 結論だけでなく導出を示す。ただし高校数学に橋渡しする。
2. **後追いできる相互リンク**: 定義・定理・記号を参照リンクで結び、「これ何だっけ」を即解消。
3. **インタラクティブな可視化**: パラメータを動かす／データを操作することで、式の意味を体感。

### 4.1 トピックの標準構成（「概念 → 操作 → 演習」）

```
│   - 数式（KaTeX）+ 導出。専門用語は [リンク] 付き │
│   - 「高校数学ではこう」という橋渡しコラム      │
├─────────────────────────────────────────┤
│ ② 操作 (Interact)                         │
│   - パラメータ/データを動かせる可視化         │

└─────────────────────────────────────────┘
```

### 4.1.5 トピック深掘りの標準: **Level 制（例外なし）** 【必須要件】

> 参照基準: [回帰分析って知ってる？―完全に理解した人へ贈る絶望の谷](https://zenn.dev/green_tea/articles/0bce4d47ab0674)

**すべてのトピックは、上記記事と同等の「Level 0〜6 の階層的深掘り」構造を持つこと。例外は許容しない。**
浅い解説で終わらせず、初学者から研究フロンティアまで段階的に潜れる設計を標準とする。
（全トピックが Level 6 まで初日に揃っている必要はないが、**構造としては必ず Level 枠を用意し、最低でも Level 0〜2 を満たす**。Level は順序ではなく多角的な観点であることを明記する。）

| Level | 位置づけ | 各トピックで満たすべき内容（回帰分析の例） |
| --- | --- | --- |
| **L0** | ブラックボックス | ツールとして使える（何の問題を解くか、どう呼び出すか） |
| **L1** | 古典的理解 | 完全な導出（例: OLS の行列微分）、幾何学的直感（射影のアニメーション） |
| **L2** | 仮定体系 | 前提条件と定理の証明（例: ガウス=マルコフ/BLUE）、仮定違反の影響と診断 |
| **L3** | 予測最適化 | バイアス-バリアンス分解、正則化（Ridge/Lasso のベイズ的再解釈）、非線形拡張 |
| **L4** | 因果 | 交絡・欠落変数バイアスの数理、識別戦略（FWL 定理、DID 等） |
| **L5** | 高次元/最先端 | 高次元理論、Post-Selection Inference、DML 等の導出と数値実験 |
| **L6** | フロンティア | 未解決・研究領域（「ここは到達途上」と正直に明示してよい） |

各 Level 内の標準セクション構成（記事に倣う）:
```
Level X: [概念名]
  ├─ 理論（直感的導入 → 数式）
  ├─ 導出・証明（折りたたみで全ステップ）
  ├─ 図解 / インタラクティブ操作（幾何学的直感）
  ├─ 応用・処世術（実務での落とし穴）
  ├─ 実装コード（任意）
  ├─ 数値実験（理論を実データ/シミュで検証）
  └─ まとめ
```

設計上の要請:
- **対象読者ガイド**を各トピック冒頭に置く（「どの Level からどう読むか」）。
- Level 間・トピック間は §4.3 の相互リンクで結び、前提（例: L4 因果は L1 回帰が前提）を辿れるようにする。
- 「完全に理解した」と思った人にこそ次の Level を提示する——深さの存在を可視化し、慢心を防ぐ。

### 4.2 数式の方針
- レンダリングは **KaTeX**（高速・PWA向き）。
- すべての数式は「結論」だけでなく **導出ステップを折りたたみ表示**できる。
- 記号・専門用語にはホバー/タップで定義ポップオーバー（用語集と連動）。

### 4.3 相互リンク / 知識グラフ
- 用語・定義・定理を **ノード** として管理し、トピック間で相互参照。
- 「この証明で使った◯◯の定義へ」を1タップで辿れる（証明の後追い）。
- ⚠️ ノードの粒度（用語単位 / 定理単位）は MVP 実装時に確定。

### 4.4 インタラクティブ可視化（肌感覚の中核）
- 「パラメータを動かす」「点を打つ/動かす」「サンプルを引き直す」操作を共通部品化。
- 計算はクライアント側で実時間実行（MVP 範囲の統計計算は JS で十分）。
- ⚠️ 重い計算（MCMC, LightGBM 学習）はフェーズ B/C で **Pyodide もしくはサーバ実行** を検討。

#### 可視化スタイルの参照基準: **『アルゴリズム図鑑』** 【必須要件】

> 参照: `/Users/michika_maruyama/Desktop/kindle_coding/screenshots_books/アルゴリズム図鑑`（全136ページのスクショ）
> 補足ノート: `/Users/michika_maruyama/Desktop/obsidian_note/05_Bookmemo/アルゴリズム図鑑`

可視化の見た目・運び方は同書のスタイルに準拠する。特徴:
- **状態をフレーム単位で見せる**: 1つの処理を「状態1 → 状態2 → …」と離散ステップに分解し、
  ユーザーが**ステップを前後に送りながら**1コマずつ観察できる（自動再生 + 手動コマ送りの両対応）。
- **色付きの要素（ブロック/ノード/矢印）**で対象を表現。比較・交換・着目している要素を**色とハイライト**で示す。
- **「解説」「補足」コールアウト**を図の近傍に配置し、そのステップで「今・何が・なぜ」起きているかを短文で添える。
- 装飾を排した**クリーンで余白の多いレイアウト**。1画面1概念。番号付き見出し（例: `No. 0-1`）で位置を明示。
- アルゴリズム/手法の説明では、まず**小さな具体例（数個の要素）**で原理を見せてから一般化する。

この「コマ送り + 色ハイライト + 近傍コールアウト」を**可視化部品の共通フォーマット**とし、
統計・ML・ベイズ・因果・時系列・（狭義の）アルゴリズム全トピックで一貫して用いる。

#### 連動レベル: **強連動（リッチ）** 【決定済み】
本アプリの中核体験。操作（スライダー/ドラッグ/再サンプリング）に対し、
**グラフだけでなく数式中の該当項の数値・ハイライトもリアルタイムに連動**させる。
「式のこの部分が、いま動いている」が視覚的に分かる状態を標準とする。

実現方針:
- 数式は KaTeX で描画し、項ごとに `id` を付与 → 操作イベントで該当項を再描画/ハイライト。
- 操作値・派生値（平均、分散、p値 等）を単一の **state ストア** に集約し、
  グラフ・数式・数値表示が同じ state を購読（single source of truth）。
- 部品は「操作系（コントロール）」「計算系（純関数）」「描画系（グラフ/数式）」を疎結合に分離し、
  トピック横断で再利用できるよう設計する。

#### MVP で用意したい可視化例（準1級基礎）
| トピック | 操作 | 得られる肌感覚 |
| --- | --- | --- |
| 正規分布 | μ, σ をスライダーで変更 | パラメータと形状の対応 |
| 中心極限定理 | 元分布を選び n とサンプリングを回す | 平均の分布が正規に近づく |
| 信頼区間 | 母集団から繰り返し標本抽出 | 「95%」の頻度論的意味 |
| 仮説検定 | 効果量・n・有意水準を変更 | p値・検出力・第一種/第二種の誤り |
| 単回帰 | 点をドラッグ / 直線の傾き切片を操作 | 最小二乗が残差平方和を最小化する様子 |
| 重回帰・正則化 | λ を変更 | 係数の縮小・バイアス分散 |

## 7. MVP 定義（最初に検証するもの）

### 目的
「概念→操作→演習」の1トピック体験が、**実際に肌感覚の理解につながるか**を検証する。

### 最初のプロトタイプ題材: **中心極限定理（CLT）** 【決定済み】
体験設計（強連動）を最初に固める題材として CLT を採用。
- 操作: 元分布を選ぶ（一様/指数/二項 など）→ 標本サイズ n を変える → サンプリングを繰り返す
- 連動: 標本平均のヒストグラムが更新されると同時に、数式
  `平均 → μ`, `標準偏差 → σ/√n` の該当項に現在の n・σ が代入・ハイライトされる
- 肌感覚: 「元分布が何であれ、n を増やすと平均の分布が正規に近づき、ばらつきが σ/√n に縮む」

### MVP に含むもの
1. 統計検定準1級・基礎から **代表トピックを 3〜5 本**（中心極限定理 ← 起点 / 正規分布 / 信頼区間 / 仮説検定 / 単回帰）
2. 各トピックで「概念（数式+導出+用語リンク）」「操作（可視化）」「演習（確認問題+即時FB）」が動く
3. アカウント認証 + 進捗/正誤履歴の保存・同期
4. PWA としてスマホ・PC両方で快適に動作

### MVP に含まないもの（後続フェーズ）
- 機械学習 / ベイズ / 因果推論コンテンツ
- 間隔反復による最適復習（簡易版に留める）
- 重い計算（MCMC・LightGBM 学習）
- ネイティブアプリ配信、CMS、課金

### MVP の完了条件（検証）
- [ ] 上記トピックを通しで学習し、操作→数式の連動が破綻なく動く
- [ ] 別端末でログインし進捗が同期される
- [ ] テストユーザーが「式の意味が腑に落ちた」と言える（定性検証）
