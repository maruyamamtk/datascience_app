# 設計案 — 歩く骨格(Walking Skeleton): 中心極限定理トピック

関連: [SPEC.md](../../SPEC.md) / [CLAUDE.md](../../CLAUDE.md)
目的: 本アプリ最大のリスク（**操作→数式の強連動** + **アルゴリズム図鑑スタイルのコマ送り可視化**）を、
中心極限定理(CLT)の1トピックを縦に貫通させて検証する。ここで確立した部品・規約を全トピックの土台とする。

---

## 1. 技術スタック（最終案）

| 領域 | 採用 | 理由 |
| --- | --- | --- |
| パッケージ管理 | **pnpm** | 高速・省ディスク |
| フレームワーク | **Next.js (App Router) + TypeScript** | Web優先→PWA、将来のSSG/SSR両対応 |
| スタイル | **Tailwind CSS** | 余白の多いクリーンなレイアウト(図鑑スタイル)を素早く実現 |
| 数式 | **KaTeX**（`\htmlId` 拡張を使用, `trust:true`/`strict:false`） | 項ごとに id 付与 → ハイライト/数値差し込みを可能にする |
| 可視化 | **D3.js**（スケール/補間/DOM操作）+ Canvas/SVG | スライダー/ドラッグ/再サンプリングの自由度 |
| 状態管理 | **Zustand**（トピック単位ストア） | 軽量・single source of truth を素直に表現 |
| コンテンツ | **MDX** | 解説文に操作部品を直接埋め込む |
| 乱数/統計 | 自前の純関数（`lib/stats`） | MVP範囲は依存少なく自前実装、テスト可能に |
| PWA | next-pwa もしくは手動 manifest + SW | オフライン閲覧・ホーム画面追加 |
| テスト | **Vitest** | 計算層(純関数)の単体テスト |

> 逸脱する場合は SPEC を更新し理由を残す(CLAUDE.md §3)。

## 2. アーキテクチャ: 3層の疎結合（全可視化共通の規約）

```
[Control層]  操作UI（スライダー/ボタン/ドラッグ）
     │ ユーザー操作 → setState
     ▼
[Store]      Zustand: 操作値 + 派生値の single source of truth
     │ subscribe
     ├──────────────┬───────────────┐
     ▼              ▼               ▼
[Compute層]    [Render層:Graph]  [Render層:Math]
 純関数(副作用なし)  D3描画         KaTeX + 項ハイライト
 単体テスト可能
```

- **Compute層は純関数**（`(params) => 派生値/フレーム列`）。副作用なし、Vitestで検証。
- 操作値・派生値（平均、σ/√n、現在のフレーム等）は**1つのストアに集約**。Graph/Math/数値表示は同じストアを購読。
- 描画層は Web に閉じ、ロジックと疎結合（将来のネイティブ化で再利用）。

## 3. 数式の強連動（核心リスク）

- KaTeX で数式を描画し、着目させたい項を `\htmlId{term-sigma}{\sigma}` のように **id 付き**で記述。
- ストアの該当値が変わると、`document.getElementById('term-sigma')` 系で**色・数値バッジ**を更新。
- 用語・定義は**ノード**として `content/terms/*.mdx` に持ち、数式項からポップオーバーで参照（証明の後追い, SPEC §4.3）。
- ⚠️ 検証ポイント: 60fps を保てるか / KaTeX 再描画コストを避け **DOMパッチで差分更新**できるか。

## 4. 可視化共通部品（アルゴリズム図鑑スタイル）

`components/viz/` に以下を用意し、全トピックで使い回す:
- **`StepPlayer`**: フレーム列を受け取り「自動再生 / 一時停止 / 1コマ前後送り / スライダー」を提供。
- **`Highlight`**: 着目要素を色・枠で強調（フレームごとに対象が変わる）。
- **`Callout`**: 図の近傍に「いま・何が・なぜ」を短文表示（"解説"/"補足"）。
- レイアウト規約: 1画面1概念、余白多め、番号付き見出し、小さな具体例から一般化。

## 5. コンテンツ構造（Level 制テンプレート）

```
content/
  topics/
    central-limit-theorem.mdx   # Level枠(L0..L6)を持つテンプレートで記述
  terms/
    sample-mean.mdx             # 用語ノード(相互リンク対象)
    standard-error.mdx
```
- トピックは §4.1.5 の Level 枠を必ず持つ（CLT は L0〜L2 を満たす）。
- 各 Level に「概念→操作→演習」を内包（§4.1）。

## 6. CLT 縦串の中身（受け入れ条件）

- **概念**: 標本平均の分布、`E[X̄]=μ`、`SD[X̄]=σ/√n` を導出付きで提示。項に id を付与。
- **操作**: 元分布を選択（一様/指数/二項）→ n をスライダー変更 → 「サンプリング」ボタンで標本平均を蓄積。
  - ヒストグラムが更新されると同時に、数式の `σ/√n` 項へ現在の n・σ が差し込まれハイライト。
  - StepPlayer で「1回ずつ標本を引く」コマ送りも可能。
- **演習**: 確認問題（例: n を4倍にすると標準誤差は何倍か）→ 即時フィードバック → 操作へ戻るリンク。
- **動作確認**: スマホ/PC 両方、操作 60fps 目標、オフライン閲覧。

## 7. スコープ外（この骨格では作らない）

- Supabase 認証・進捗保存（体験が腑に落ちると確認後に着手）
- CLT 以外のトピック、重い計算（MCMC/LightGBM）、ネイティブ配信

## 8. ディレクトリ構成（初期案）

```
  topics/[slug]/page.tsx
components/

lib/
  stats/             # 乱数・分布・標本平均などの純関数(テスト対象)
  store/             # Zustand ストア(トピック単位)
content/
  topics/ terms/     # MDX
public/               # manifest, icons
```
